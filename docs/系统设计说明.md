# 系统设计说明

## 1. 系统目标

大田无人机巡检监控系统用于集中展示无人机巡检画面、烟田病虫害检测统计、实时链路状态、告警事件和数据分析入口。当前版本是基础框架，先完成前后端通信、监控大屏布局、数据库初始化和模拟数据服务。

## 2. 总体架构

- WPF 桌面端负责监控大屏展示，通过 REST API 拉取初始数据，并预留 WebSocket 实时刷新。
- FastAPI 后端负责统一提供系统状态、视频状态、检测统计、告警列表、系统配置和实时推送数据。
- SQLite 作为开发阶段数据库，SQLAlchemy 作为 ORM 层，后期通过 `DATABASE_URL` 切换到 MySQL。

## 3. 模块划分

- 视频监控模块：展示无人机视频占位画面、分辨率、帧率和链路延迟。
- 检测统计模块：展示总检测数、当前分钟检测数和风险指数。
- 告警模块：展示告警时间、类型、内容、置信度和处理状态。
- 快捷入口模块：预留直播详情、系统诊断、配置管理、病虫害预测、烟田数字孪生、数据分析。
- 实时通信模块：通过 `/ws/realtime` 周期推送聚合大屏数据。

## 4. 数据流

1. 后端启动时初始化数据库并写入基础配置、告警和检测统计种子数据。
2. WPF 启动后每 2 秒调用 REST API 更新界面。
3. WebSocket 连接可用时，前端接收 `/ws/realtime` 推送并覆盖 REST 轮询数据。
4. 后端服务不可用时，WPF 保留本地默认数据并显示离线状态。

## 5. 后续扩展
