# 系统设计说明

## 1. 系统目标

大田无人机巡检监控系统用于集中展示无人机巡检画面、烟田病虫害检测统计、实时链路状态、告警事件和数据分析入口。当前版本除了桌面大屏，还新增了 Web 分析大屏，支持图片上传、视频抽帧分析、实时流采样分析、热力分割结果展示和历史记录回看。

## 2. 总体架构

- WPF 桌面端负责本地监控大屏展示，通过 REST API 拉取初始数据，并预留 WebSocket 实时刷新。
- React Web 大屏负责图像展示、模型分析、热力分割结果查看、历史结果筛选和趋势图展示。
- FastAPI 后端负责统一提供系统状态、视频状态、检测统计、告警列表、系统配置、分析任务、结果存储和实时推送数据。
- SQLite 作为开发阶段数据库，SQLAlchemy 作为 ORM 层，后期通过 `DATABASE_URL` 切换到 MySQL。

## 3. 模块划分

- 视频监控模块：展示无人机视频占位画面、分辨率、帧率和链路延迟。
- 检测统计模块：展示总检测数、当前分钟检测数和风险指数。
- 告警模块：展示告警时间、类型、内容、置信度和处理状态。
- 快捷入口模块：预留直播详情、系统诊断、配置管理、病虫害预测、烟田数字孪生、数据分析。
- 实时通信模块：通过 `/ws/realtime` 周期推送聚合大屏数据。
- 模型推理模块：通过通用 `ModelRunner` 接口接收 RGB 图像，输出概率图、二值掩码和置信度摘要。
- 分析任务模块：支持图片、视频和实时流三类输入，生成原图、热力分割图、掩码图和统计结果。

## 4. 数据流

1. 后端启动时初始化数据库并写入基础配置、告警和检测统计种子数据。
2. Web 大屏上传图片或视频，或提交实时流地址后，后端创建分析任务并落库。
3. 模型推理链路输出概率图、掩码图、热力叠加图和杂草覆盖面积占比、像素面积、估算植株数等统计值。
4. Web 大屏通过 `/api/analysis/*` 接口读取任务、结果和历史记录，并通过 `/ws/analysis/{job_id}` 接收任务进度与最新分析帧。
5. `/ws/realtime` 继续提供系统级状态，并附带最近一次分析摘要供首页总览使用。
