# 大陆服务器学生版上线清单

这份清单按下面的目标设计：

- 预算控制在 300 到 400 元每年
- 服务器部署在中国大陆
- 小规模项目使用
- 尽量接近当前本地运行效果
- 域名审核和备案期间，先通过公网 IP 完成部署和测试

## 一、购买清单

推荐方案：

- 腾讯云轻量应用服务器，2 核 4G，1 年
- `.cn` 域名 1 个
- 备案通过后上线

预算估算：

- 服务器：约 200 到 260 元/年
- 域名：约 40 到 60 元/年
- SSL 证书：0 元，使用 Let's Encrypt
- DNS：0 元，使用云厂商自带解析
- 合计：约 250 到 320 元/年

如果你想更正式一点，也可以把域名换成 `.com`：

- 域名：约 80 到 100 元/年
- 合计：约 290 到 360 元/年

## 二、购买时怎么选

服务器：

- 地域：优先选离你近的大陆节点，比如上海、广州、北京
- 系统：Ubuntu 22.04 LTS 或 Ubuntu 24.04 LTS
- 配置：2 核 4G
- 带宽：5M 足够起步
- 数据盘：默认即可

域名：

- 预算优先：`.cn`
- 更通用：`.com`
- 尽量选简短、好记、和项目相关的名字

## 三、备案前要准备的东西

- 你自己的身份证
- 能接收短信和电话的手机号
- 已经实名认证的云账号
- 已经购买好的大陆服务器
- 已经购买并完成实名认证的域名

## 四、服务器上线前要做的代码准备

### 1. 后端生产环境变量

在服务器上创建 `backend/.env`，可以参考：

- [backend/.env.production.example](/E:/Uav-Field-Inspection-System/backend/.env.production.example)

重点改这些值：

- `APP_ENV=production`
- `MODEL_PATH=/srv/uav-field-inspection-system/backend/models/best.ckpt`
- `MODEL_BACKEND=ckpt`

### 2. 前端生产环境变量

在服务器上构建前端前，创建：

- `frontend/uav-inspection-web/.env.production`

参考：

- [frontend/uav-inspection-web/.env.production.example](/E:/Uav-Field-Inspection-System/frontend/uav-inspection-web/.env.production.example)

第一阶段先填服务器公网 IP，例如：

```env
VITE_API_BASE_URL=http://123.123.123.123
VITE_WS_BASE_URL=ws://123.123.123.123
```

等域名审核完成并启用 HTTPS 后，再改成：

```env
VITE_API_BASE_URL=https://uav-demo.cn
VITE_WS_BASE_URL=wss://uav-demo.cn
```

## 五、服务器部署步骤

### 1. 安装基础环境

```bash
sudo apt update
sudo apt install -y git nginx python3 python3-venv python3-pip nodejs npm
```

如果系统自带 Node 版本太旧，改用 NodeSource 安装 Node 20。

### 2. 拉代码

```bash
sudo mkdir -p /srv
cd /srv
sudo git clone https://github.com/Misako001/uav-field-inspection-system.git
sudo chown -R $USER:$USER /srv/uav-field-inspection-system
cd /srv/uav-field-inspection-system
```

### 3. 上传模型文件

把你的模型权重上传到：

```text
/srv/uav-field-inspection-system/backend/models/best.ckpt
```

### 4. 启动后端

```bash
cd /srv/uav-field-inspection-system/backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.production.example .env
```

然后按实际服务器路径修改 `.env`。

### 5. 构建前端

```bash
cd /srv/uav-field-inspection-system/frontend/uav-inspection-web
npm install
cp .env.production.example .env.production
npm run build
```

### 6. 配置 systemd

参考：

- [deploy/systemd/uav-backend.service](/E:/Uav-Field-Inspection-System/deploy/systemd/uav-backend.service)

先创建服务用户：

```bash
sudo useradd -r -s /usr/sbin/nologin uav
sudo chown -R uav:uav /srv/uav-field-inspection-system/backend
```

再把 service 文件复制到：

```text
/etc/systemd/system/uav-backend.service
```

启动：

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now uav-backend
sudo systemctl status uav-backend
```

### 7. 配置 Nginx

参考：

- [deploy/nginx/uav-inspection.conf](/E:/Uav-Field-Inspection-System/deploy/nginx/uav-inspection.conf)

第一阶段可以直接保持：

```nginx
server_name _;
```

这样可以直接通过公网 IP 访问。

等域名审核和解析都完成后，再改成你的真实域名，例如：

```nginx
server_name uav-demo.cn www.uav-demo.cn;
```

然后复制到：

```text
/etc/nginx/conf.d/uav-inspection.conf
```

检查并重启：

```bash
sudo nginx -t
sudo systemctl restart nginx
```

## 六、先用公网 IP 测试，再切换到域名和 HTTPS

第一阶段先测试：

- `http://服务器公网IP`
- `http://服务器公网IP/api/system/status`
- `http://服务器公网IP/docs`

只要这些地址能打开，说明部署已经跑通。

第二阶段在域名审核完成后做：

- 域名 DNS 解析到服务器公网 IP
- Nginx `server_name` 改成域名
- 前端 `.env.production` 改成 `https://域名` 和 `wss://域名`
- 重新执行 `npm run build`
- 重启 Nginx

第三阶段在备案完成后启用 HTTPS：

推荐使用 Certbot：

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d example.cn -d www.example.cn
```

成功后，前端访问地址会从 `http` 变成 `https`，WebSocket 也会自动走 `wss`。

## 七、上线后的验证清单

- `http://服务器公网IP` 能打开首页
- `http://服务器公网IP/api/system/status` 能正常返回
- `http://服务器公网IP/docs` 能打开后端文档
- 域名能正常解析到服务器公网 IP
- `http://你的域名` 能打开首页
- `https://你的域名` 能打开首页
- 页面顶部显示后端在线
- 上传图片可以出结果
- `/docs` 能打开 FastAPI 文档

## 八、你这个项目当前最适合的上线边界

这套学生预算版更适合：

- 小规模演示
- 课程项目展示
- 少量图片分析
- 少量视频分析

如果你以后要长期对外提供服务，或者要支撑多人并发、实时流推理，后续建议升级：

- 服务器内存到 8G 或更高
- 数据库改 MySQL
- 流媒体服务单独部署
- 模型推理迁移到 GPU 服务器
