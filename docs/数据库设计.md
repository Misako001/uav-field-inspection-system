# 数据库设计

## 1. 数据库策略

开发阶段默认使用 SQLite：`sqlite:///./uav_inspection.db`。后期切换 MySQL 时，将 `.env` 中的 `DATABASE_URL` 改为 MySQL 连接串即可：

```env
DATABASE_URL=mysql+pymysql://uav_user:uav_password@127.0.0.1:3306/uav_inspection
```

## 2. 表结构

### alert_events

告警事件表。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | Integer | 主键 |
| occurred_at | DateTime | 告警发生时间 |
| alert_type | String | 告警类型 |
| content | Text | 告警内容 |
| confidence | Float | 置信度 |
| status | String | 处理状态 |
| created_at | DateTime | 创建时间 |

### detection_statistics

检测统计快照表。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | Integer | 主键 |
| total_count | Integer | 总检测数 |
| current_minute_count | Integer | 当前分钟检测数 |
| risk_index | Float | 风险指数 |
| recorded_at | DateTime | 记录时间 |

### system_configs

系统配置表。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | Integer | 主键 |
| key | String | 配置键 |
| value | Text | 配置值 |
| description | String | 配置说明 |

### analysis_jobs

分析任务主表。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | Integer | 主键 |
| source_type | String | 来源类型：image / video / stream |
| source_name | String | 原始文件名或流地址 |
| source_uri | Text | 流地址 |
| source_media_path | Text | 上传原文件路径 |
| status | String | pending / running / completed / failed / stopped |
| progress | Float | 任务进度 |
| model_backend | String | 模型后端标识 |
| frame_count | Integer | 已处理样本数或帧数 |
| average_coverage_ratio | Float | 平均杂草覆盖面积占比 |
| estimated_plant_count | Integer | 估算植株数 |
| average_confidence | Float | 平均置信度 |
| latest_result_id | Integer | 最近结果 ID |
| error_message | Text | 错误信息 |
| created_at | DateTime | 创建时间 |
| started_at | DateTime | 开始时间 |
| completed_at | DateTime | 完成时间 |

### analysis_results

分析结果摘要表。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | Integer | 主键 |
| job_id | Integer | 所属任务 |
| source_image_path | Text | 原图或摘要帧路径 |
| heatmap_image_path | Text | 热力分割图路径 |
| mask_image_path | Text | 二值掩码图路径 |
| thumbnail_path | Text | 缩略图路径 |
| weed_coverage_ratio | Float | 杂草覆盖面积占比 |
| weed_pixel_area | Integer | 杂草像素面积 |
| estimated_plant_count | Integer | 基于连通域的估算植株数 |
| average_confidence | Float | 平均置信度 |
| processing_time_ms | Integer | 单次处理耗时 |
| result_time | DateTime | 结果生成时间 |
| summary_note | Text | 结果摘要说明 |

### analysis_frames

视频 / 实时流逐帧结果表。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | Integer | 主键 |
| job_id | Integer | 所属任务 |
| frame_index | Integer | 抽帧序号 |
| frame_timestamp_seconds | Float | 视频时间点或采样秒数 |
| source_frame_path | Text | 原始帧路径 |
| heatmap_image_path | Text | 热力分割图路径 |
| mask_image_path | Text | 二值掩码图路径 |
| weed_coverage_ratio | Float | 当前帧杂草覆盖面积占比 |
| weed_pixel_area | Integer | 当前帧杂草像素面积 |
| estimated_plant_count | Integer | 当前帧估算植株数 |
| average_confidence | Float | 当前帧置信度 |
| created_at | DateTime | 创建时间 |

## 3. 初始化数据

后端启动时执行 `init_db()`，自动创建表，并在空库中写入视频流地址、刷新间隔、数据库类型、示例告警和检测统计数据。分析任务、结果和帧记录在运行时按需写入。
